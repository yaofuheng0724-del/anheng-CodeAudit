# 深度审计页面简化设计

**日期**: 2026-05-31
**状态**: 已确认

---

## 问题背景

用户反馈深度审计页面不易理解，主要有两个问题：
1. **启动屏幕太复杂**：终端命令输入方式不直观，用户不知道要输入 "audit" 才能开始
2. **整体流程不清楚**：进入页面后不知道审计任务如何执行、各阶段做什么

---

## 设计方案

采用**完全简化方案**：移除所有赛博朋克效果，改为简洁现代的设计风格。

---

## 第一部分：SplashScreen（启动屏幕）

### 改动前
- 赛博朋克风格终端界面
- 用户需要在终端输入 "audit" 命令才能开始
- 包含大量视觉效果：cyber grid、scan line、CRT effect、glitch overlay、data stream、floating hex codes 等

### 改动后
简洁的欢迎页面：

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                    DeepAudit                            │
│           Autonomous Security Agent                     │
│                                                         │
│     ┌─────────────────────────────────────────┐        │
│     │                                         │        │
│     │    🔍 开始审计任务                       │        │
│     │                                         │        │
│     └─────────────────────────────────────────┘        │
│                                                         │
│           点击按钮创建并启动新的安全审计任务            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 设计要点
- **背景**：纯色或简单渐变（浅灰/深色模式自适应），使用 `bg-background` 类
- **Logo**：
  - 主标题：DeepAudit（主题色）
  - 副标题：Autonomous Security Agent
  - 可使用 Shield 图标作为装饰
- **按钮**：
  - 文字：「开始审计任务」
  - 风格：醒目大按钮，主题色（primary），带图标
  - 行为：点击直接打开 CreateAgentTaskDialog
- **提示文字**：按钮下方一行简短说明
- **移除内容**：
  - 所有赛博朋克效果
  - 终端窗口
  - boot sequence 动画
  - 命令输入功能
  - 所有 CSS 动画

## 第二部分：PhaseDetail 引导卡片

### 改动
在左侧日志区（PhaseDetail）顶部添加可折叠的阶段引导卡片。

### 展开状态
```
┌─────────────────────────────────────────────────────────┐
│ ▼ 任务执行指南                                    [折叠] │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 审计任务将自动依次执行以下5个阶段，您可以观察实时进度：  │
│                                                         │
│ 📦 准备 - 克隆仓库、初始化沙箱、构建代码索引            │
│ 🔍 侦察 - 项目结构分析、技术栈识别、入口点发现          │
│ 🔬 分析 - 深度代码审计、漏洞检测                        │
│ ✅ 验证 - 漏洞验证、PoC 生成                            │
│ 📊 报告 - 结果汇总、评分计算                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 折叠状态
```
┌─────────────────────────────────────────────────────────┐
│ ▶ 任务执行指南                                    [展开] │
└─────────────────────────────────────────────────────────┘
```

### 设计要点
- **位置**：PhaseDetail 组件顶部，在"当前阶段标题"区域之前
- **状态**：默认展开，用户可点击标题栏折叠/展开
- **标题栏**：
  - 左侧：折叠图标（ChevronDown/ChevronRight）
  - 中间："任务执行指南"
  - 右侧：可选的折叠按钮文字
- **内容**：
  - 一行说明文字
  - 5个阶段的简洁列表（图标 + 名称 + 说明）
  - 使用现有 `AUDIT_PHASE_CONFIG` 数据保持一致性
- **样式**：
  - 浅色背景卡片（`bg-muted/50`）
  - 边框（`border-border`）
  - 圆角（`rounded-lg`）
  - 不占用太多垂直空间
- **行为**：
  - 点击标题栏切换折叠/展开
  - 平滑动画过渡（`transition-all duration-200`）

## 验收标准

1. 启动页面简洁明了，用户一眼就能看到「开始审计任务」按钮
2. 点击按钮直接打开创建任务对话框，无需输入命令
3. 审计页面左侧顶部有引导卡片，默认展开
4. 引导卡片内容清晰说明5个阶段的作用
5. 用户可以折叠引导卡片，折叠状态只显示标题栏
6. 折叠/展开有平滑动画过渡
